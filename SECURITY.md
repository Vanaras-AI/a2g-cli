# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Email **security@vanaras.ai** with:

- A description of the vulnerability.
- Steps to reproduce or a proof of concept.
- The potential impact.

## What Qualifies

- Authentication or authorization bypasses.
- Signature verification flaws.
- Audit ledger integrity issues.
- Policy enforcement bypasses.
- Injection or path traversal in CLI inputs.

## Response Timeline

- **48 hours** -- acknowledgment of your report.
- **7 days** -- initial assessment and severity classification.
- **30 days** -- target for a fix or mitigation in a new release.

We will credit reporters in the release notes unless anonymity is requested.
